This software uses machine learning to interpret game frames. Most artificial intelligence models for games contain billions of parameters and require massive servers to function. SauerkrautLM-Doom-MultiVec operates differently. It utilizes a highly optimized design that fits 1.3 million parameters into a tiny package.

This design choice allows the model to react to enemies and obstacles without input lag. You see the character move, shoot, and navigate corridors based on the decisions of the model. 

## 🔧 Frequently Asked Questions

### Can I control the character?
The software defaults to autonomous mode. It plays the game for you. You can press the Escape key to pause the simulation or take manual control if you want to intervene.

### Does this require an internet connection?
No. Once you download the model files, the software runs entirely on your local machine. You do not need to be online to play.

### Is this safe to use?
Yes. The software runs in a contained environment. It interacts only with the game files you specify during the initial setup.

### The game runs too fast. Can I slow it down?
Yes. Open the settings menu inside the application. You will find a slider labeled Game Speed. Adjust this to match your preference for viewing the model performance.

## 🛠️ Advanced Configuration

For users who want to experiment, the settings folder contains a configuration file. You can open this file with a basic text editor like Notepad. 

- `view_mode`: Change this to toggle between the model's vision and the standard game view.
- `save_logs`: Set this to true if you want the software to record performance data.
- `max_fps`: Use this to cap the frame rate if your hardware runs the game too quickly.

Save the file after you make changes. Restart the application to apply the new settings.

## 🔍 Troubleshooting Performance

If you experience issues, consider these solutions:

- Verify game files: Ensure your DOOM.WAD file is not corrupted or missing. 

The software keeps a log file in the installation directory. If the software crashes, this log explains the specific error. You can share this file if you seek help from the community.
